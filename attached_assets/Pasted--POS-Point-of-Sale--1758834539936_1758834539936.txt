Создайте профессиональный, современный веб-POS-терминал (Point of Sale) и систему управления инвентарем. Проект должен быть написан на Python (Flask) с использованием SQLite для хранения данных.

Основная цель: обеспечить быстрое и безошибочное обслуживание кассиров и предоставить администраторам мощные инструменты для анализа.

1. Расширенный модуль кассового обслуживания (POS)
Скорость и удобство:

Мгновенный поиск: Реализуйте мгновенный поиск товаров (типа "live search" или "autocomplete") по названию или артикулу. Поиск должен работать очень быстро, чтобы кассир не ждал.

Горячие клавиши (Shortcuts): Включите поддержку основных горячих клавиш для ускорения работы (например, F1 для поиска, F2 для оплаты, F3 для скидки).

Разделение чека: Добавьте возможность частичной оплаты или разделения чека между разными способами оплаты (например, часть наличными, часть картой).

Автоматические скидки: Реализуйте применение скидок по условиям (например, "скидка 10% на все товары из категории 'Молочные продукты'" или "скидка 5% при покупке от 5000 руб.").

Отложенные чеки: Возможность отложить текущий чек и начать новый, чтобы быстро обслужить следующего клиента.

2. Продвинутое управление товарами и инвентарем
Детализация данных:

Единицы измерения: Добавьте поле для указания единицы измерения товара (шт., кг., литр и т.д.).

Изображения товаров: Возможность загрузки небольшого изображения для каждого товара. Отображайте это изображение на экране поиска POS-терминала.

Управление поставщиками: Создайте отдельный модуль для регистрации поставщиков и привязки товаров к конкретному поставщику.

Управление остатками:

Минимальный остаток: Добавьте поле "Минимальный остаток". Если фактический остаток падает ниже этого значения, система должна выводить уведомление на дашборде администратора.

Приходные накладные: Внедрите модуль для регистрации приходных накладных. Товар должен поступать на склад не просто "добавлением количества", а через оформление документа прихода с указанием поставщика и закупочных цен.

3. Расширенная аналитика и отчетность
Отчет по прибыли: Создайте отчет, который рассчитывает чистую прибыль (Выручка - Себестоимость) за выбранный период.

ABC-анализ (начальный уровень): Выводите список топ-10 самых продаваемых товаров (А-товары) и топ-10 наименее продаваемых товаров (С-товары) по количеству и по выручке.

Графики: На дашборде администратора используйте простую библиотеку JavaScript (например, Chart.js), чтобы отобразить график динамики выручки за последние 7 дней.

Технические требования и UI
Технологии: Python Flask, SQLite, HTML/CSS/JavaScript.

Адаптивность: Сделайте интерфейс POS-терминала максимально адаптивным для работы как на ПК, так и на планшете (мобильном устройстве).

UI/UX: Используйте Bootstrap или другой CSS-фреймворк для создания современного и чистого интерфейса с четкой иерархией. Интерфейс кассира должен быть контрастным и содержать минимум лишних элементов.
